# opdeck

**Пульт управления сессиями ИИ-агентов. Честный PTY поверх нативных CLI, без аккаунта, local-first.**

Много промтов, много веток кода — одна панель управления. **opdeck** — это супервизор для Claude Code, Codex, Qwen, Grok, opencode и других CLI-агентов на вашей машине. Видно, что агент делает в реальном времени: какие файлы трогает, в каких папках работает, какие ошибки выдаёт. Параллельные сессии в изолированных ветках кода. Слияние результатов в один клик. Без облака.

## Что это

Диспетчерская для ИИ-кода.

Открываете **opdeck** — видите несколько воркспейсов (проектов). Внутри каждого может крутиться несколько **сессий** (агентов): Claude Code в одном терминале пишет модуль, Codex в другом рефакторит, Qwen в третьем ищет баги. Каждый работает в своей ветке кода (через `git worktree`). Статус каждого обновляется в реальном времени:

- 🔵 **синий** — агент работает;
- 🟡 **жёлтый** — ждёт вашего ответа;
- 🟢 **зелёный** — закончил;
- 🔴 **красный** — ошибка;
- ⚪ **серый** — неактивен.

Нажали Enter в жёлтой сессии — синий огонёк загорается сразу (мгновенная обратная связь, а не ожидание события агента). Агент закончил — видите список тронутых файлов, diff каждого изменения, решаете, что принять в основную ветку.

**Тёплый локальный PTY**, а не облачный холодный старт на 90 секунд.

## Чем отличается

- **Честный PTY поверх нативных CLI**, не SDK-обёртка. Агент остаётся собой: сохраняет resume, запускает свои хуки и skills, живёт в своей экосистеме (`~/.claude/`, конфиг Codex и т.д.). Никакой переделки CLI под чужой API.
- **Параллельные сессии** — разные агенты в разных ветках одновременно, без конфликтов.
- **Без аккаунта.** Ни логина, ни облака, ни трекинга. Данные на диске рядом с кодом.
- **Кросс-вендорность.** Один проект — несколько разных CLI сразу (Claude, Codex, Qwen, Grok, opencode, свой CLI).
- **Суверенитет модели.** Вы вызываете тот CLI, что скачали сами. Вендор не подменит и не заблокирует его на лету.
- **Local-first.** Терминалы, редактор, дерево файлов, слияние веток, ревью — всё на машине.

## Возможности

### Воркспейсы и сессии
- Воркспейс на проект; группы в сайдбаре (сворачиваются, цветные).
- Цветные статусы вокруг сессии + системные уведомления (агент закончил / ждёт / упал).
- Мгновенный синий статус при вводе ответа.

### Сессии в изолированных ветках
- **Git worktree** на сессию (`deck/<имя>-<id>`) — параллельная работа без конфликтов.
- **Хирургический merge-back** в `main` — выбираете файлы/куски, не весь дифф разом.

### Редактор и файлы-в-работе
- **Monaco-редактор** (как в VS Code) + дерево файлов с контекстным меню.
- **Список тронутых файлов** + git-статусы в дереве + **diff** (было → стало) по каждому файлу.

### Очередь ревью
- Вкладка **«Ревью»**: изменения по всем сессиям в одном месте, дифф в клик.
- **Предупреждение «две сессии правят один файл»** (риск конфликта).
- Отметка **«что нового с последнего взгляда»** + **прогон тестов** проекта из пульта (`npm test` / `cargo test` / `make test`).

### Расходы и лимиты
- Виджет **остатка квоты** CLI (реальные `rate_limits` Claude/Codex — как в терминале), окна 5ч/7д, без ручного ввода.

### Сторож (watchdog)
- **«Убить всё»** + **рестарт** зависшей сессии в один клик.
- **Лимит числа веток + контроль диска**; уборка ветки с диска.
- Мягкий **детект зацикливания** агента.

### Сохранение сессий (resume)
- Закрытие окна прячет (не гасит) — агенты живут в фоне.
- При полном выходе/перезагрузке — восстановление через нативный `resume` CLI (продолжает разговор).

### Режим доверия CLI
- На каждый CLI: насколько свободно агент действует без подтверждений (Claude `--permission-mode`; Codex approval/sandbox) — применяется к Deck-сессиям, личные конфиги CLI не трогаются.

### Горячие клавиши
| Клавиши | Действие |
|---|---|
| `Ctrl+N` | новый воркспейс |
| `Ctrl+1…9` | переключение воркспейсов |
| `Ctrl+Shift+U` | прыжок к ждущему/работающему агенту |

## Установка

### Быстрая (скрипт)

```bash
curl -fsSL https://raw.githubusercontent.com/222dotcrypto/opdeck/main/install.sh | bash
```

Скачивает последний релиз `.dmg`, ставит `opdeck.app` в `/Applications` и снимает карантин (релизы подписаны ad-hoc).

### Из исходников

**Требования:** macOS 11+, [Rust](https://rustup.rs/), [Node.js 18+](https://nodejs.org/), Git.

```bash
git clone https://github.com/222dotcrypto/opdeck.git
cd opdeck
npm install
npm run build:web     # собрать фронтенд (Vite → dist-web/)
npm run tauri:build   # собрать opdeck.app + .dmg
```

Готовый `.dmg` — в `src-tauri/target/release/bundle/dmg/`. Разработка с горячей перезагрузкой: `npm run tauri:dev`.

## Стек

Tauri v2 (Rust-бэкенд) · React 18 + TypeScript (Vite) · терминалы `@xterm/xterm` + `node-pty` · редактор Monaco · состояние zustand.

## Поддерживаемые CLI

Claude Code · Codex (OpenAI) · Qwen · Grok · opencode · Gemini (базовая) · свой CLI (имя + команда запуска).

## Статус

Ранняя версия (**v0.1.0**), только **macOS** (Intel + Apple Silicon). Основные фичи работают, возможны баги.

**В планах:** хуки статуса для Qwen/Grok/opencode (сейчас — Claude + Codex), bootstrap-обустройство веток (копия `.env`/порт), Windows/Linux, лучшая поддержка облачных агентов.

## English

**opdeck** is a control panel for AI agent sessions on macOS — a true PTY over native CLIs, no account, local-first.
